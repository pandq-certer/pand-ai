# 系统指标术语说明

本文档解释华北数据库团队资源规划系统中使用的专业术语和指标。

---

## 核心概念

### 1. FTE（Full-Time Equivalent）- 全职当量

**定义：**
FTE 是衡量工作量的标准单位，表示一个全职员工的工作量。

**计算方式：**
- **1.0 FTE** = 1 个全职员工 100% 的工作时间
- **0.5 FTE** = 半职员工 50% 的工作时间
- **2.0 FTE** = 2 个全职员工的工作量

**系统中的应用：**
- 周期：以**周**为单位
- 1.0 FTE = 一个人全职工作一周（5个工作日）
- 例如：某人在 A 项目投入 0.3 FTE，表示他每周有 30% 的时间用于该项目

**示例：**
```
张三本周的分配情况：
- 项目A：0.3 FTE（30% 时间）
- 项目B：0.5 FTE（50% 时间）
- 项目C：0.2 FTE（20% 时间）
总计：1.0 FTE（100% 满负荷）
```

## 投入相关指标

### 2. 投入占比（Allocation Percentage）

**定义：**
某成员在特定项目中的投入占其**总工作时间**的百分比。

**计算公式：**
```
投入占比 = (在该项目的投入量 / 在所有项目的总投入量) × 100%
```

**示例：**
```
李四本周分配情况：
- 数据库迁移项目：0.3 FTE
- 系统优化项目：0.7 FTE
- 总投入：1.0 FTE

投入占比：
- 数据库迁移项目：(0.3 ÷ 1.0) × 100% = 30%
- 系统优化项目：(0.7 ÷ 1.0) × 100% = 70%
```

**用途：**
- 帮助项目经理了解成员在各个项目上的精力分配
- 识别是否有成员过度分散在多个项目

## 可用性相关指标

### 3. 本周投入 / 本周剩余

**本周投入：**
成员本周在所有项目中已分配的工作总量（单位：FTE）。

**本周剩余：**
成员本周还有多少可用时间（单位：FTE）。

**计算公式：**
```
本周剩余 = 1.0 - 本周投入
```

**状态判断：**
- **剩余 > 0**：成员本周有空闲时间，可以接收新任务
- **剩余 = 0**：成员本周已满负荷，无法接收新任务
- **剩余 < 0**：成员过度分配（这种情况应该避免）

**示例：**
```
王五本周情况：
- 已分配项目A：0.6 FTE
- 已分配项目B：0.3 FTE
- 本周投入：0.9 FTE
- 本周剩余：1.0 - 0.9 = 0.1 FTE（还有 10% 时间可用）

赵六本周情况：
- 已分配项目：1.0 FTE
- 本周剩余：0 FTE（已满负荷，不可用）
```

## 项目相关指标

### 4. 项目负载分布

**定义：**
展示某个项目在整个团队中的资源分配情况。

**统计维度：**
- 项目总投入：所有成员在该项目的投入之和
- 成员数量：参与该项目的成员人数
- 平均投入：每个成员的平均投入量

**示例：**
```
"数据库升级项目" 负载分布：
- 张三：0.5 FTE
- 李四：0.3 FTE
- 王五：0.7 FTE
项目总投入：1.5 FTE
参与人数：3 人
```

## 时间维度指标

### 5. 周期（Planning Cycle）

**系统时间维度：**
- **基本单位**：周（Week）
- **规划周期**：4 周（当前周 + 未来 3 周）

**日期范围：**
- 始终以**周一**为一周的开始
- 动态计算：当前周的周一到未来第 4 周的周日

**示例：**
```
假设今天是 2026-03-11（周三）
规划周期：
- 第1周：03-09（周一）~ 03-15（周日）- 当前周
- 第2周：03-16（周一）~ 03-22（周日）
- 第3周：03-23（周一）~ 03-29（周日）
- 第4周：03-30（周一）~ 04-05（周日）
```

## 邮件报告中的指标

### 6. 邮件中的数据说明

**项目资源分配表：**
- **项目名称**：项目基本信息
- **成员**：参与该项目的团队成员
- **投入占比**：该成员在此项目的时间占比（相对其总工作时间）

**团队成员可用性表：**
- **本周投入**：本周所有项目的总投入（百分比形式）
- **本周剩余**：本周可用的剩余时间（百分比形式）
- **下周投入**：下周所有项目的总投入
- **下周剩余**：下周可用的剩余时间

**数据解读：**
```
| 成员 | 本周投入 | 本周剩余 | 下周投入 | 下周剩余 |
|------|----------|----------|----------|----------|
| 张三 | 60%      | 40% ✓    | 100%     | 0% ✗     |
| 李四 | 100%     | 0% ✗     | 30%      | 70% ✓    |

说明：
- ✓ 绿色表示有可用时间
- ✗ 红色表示已满负荷
- 张三本周有空闲，下周满负荷
- 李四本周满负荷，下周有空闲
```

## 常见问题

### Q1: 为什么用 1.0 表示 100%？
**A:** 这是国际标准的人力资源计量单位。1.0 FTE 代表一个全职员工的标准工作时间，便于计算和比较工作量。

### Q2: 投入量可以是小数吗？
**A:** 可以。例如 0.5 FTE 表示半职工作，0.25 FTE 表示每周 1 天的工作量。

### Q3: 为什么总和不等于 1.0？
**A:** 理论上每个人的周投入总和应该等于 1.0（100%）。如果小于 1.0，说明还有空闲时间；如果大于 1.0，说明过度分配（需要调整）。

### Q4: 邮件中的百分比和 FTE 是什么关系？
**A:**
- 邮件中显示的百分比 = FTE × 100%
- 例如：0.3 FTE = 30%，1.0 FTE = 100%

### Q5: "4 周总计 FTE" 是什么意思？
**A:** 表示某成员在 4 周规划周期内的总投入量。例如 3.2 FTE 相当于 3.2 个全职工作周的工作量。

## 指标使用建议

### 对于项目经理：

1. **关注投入占比**：确保核心成员不会被过度分散在多个项目
2. **监控可用性**：提前识别资源瓶颈，合理安排新项目
3. **平衡负载**：避免某些成员过度分配而其他成员闲置

### 对于团队负责人：

1. **总体利用率**：团队整体投入率应该接近 1.0 FTE（100%）
2. **长期规划**：查看 4 周趋势，提前进行人员调配
3. **特殊情况**：保留一些 buffer（剩余容量）应对紧急任务
